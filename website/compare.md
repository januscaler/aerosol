# How Aerosol is different

This is a high-level, honest comparison for shoppers. Products change over time — always read each vendor’s own docs and licenses. Aerosol is a young open-source project; maturity and polish will differ from paid suites.

| Topic | Aerosol | Typical “optimizer” suites | OS storage settings |
| --- | --- | --- | --- |
| Source & auditability | **Open source** — rules and UI are inspectable | Usually closed source | N/A (part of OS) |
| Pricing model | **Free**; you build or download binaries | Often subscription or paid upgrades | Included with OS |
| Where data goes | **Local scans**; no Aerosol account required | Varies; some products phone home | Local |
| Risk transparency | **Safe / review** grouping + dry run | Often one-click “clean” with less detail | Limited to what the OS exposes |
| Developer / power-user clutter | **Caches, tooling, plugins** | May focus on consumer junk files | Rarely specialized for dev stacks |
| File recovery / rescue | **Read-only folder scans**, previews, copy-out to another path (see [blog](/blog/file-recovery-mode)) | Some suites bundle separate “undelete” tools (often paid); scope varies | OS tools rarely match this workflow in one place |
| Undo / safety | **Trash option** + explicit selection; recovery never writes to source | Varies; backups still recommended | Depends on feature |
| Maturity | **Early**; expect rough edges | Often years of QA and support | Production OS feature |

Mentions of other products are for comparison only; trademarks belong to their owners. BleachBit, CCleaner, CleanMyMac, and similar tools each have different scopes — use the one that matches your threat model and comfort with closed-source maintenance utilities.
